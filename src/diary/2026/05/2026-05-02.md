今天把 mewmoire 从 GitHub 迁到 Tangled，部署改成 Tangled CI → Wrangler 直接上传 _site。看起来是修 GitHub 被 flag 的 bug，其实是主动搬家，把资产控制权抓回自己手里喵。

Tangled 基于 AT Protocol，和 Bluesky 同源。核心是把身份、代码、构建分开：账号用 DID，域名自定义；代码存 Personal Data Server；Pipeline 在自建 Spindle 上跑。这次在 Mac mini 跑本地 Spindle，Cloudflare Tunnel 把 spindle.onev.dev 指向本机 6555 端口，把 onev.cat 设为 owner，claw、paw、tail 加为成员——三姐妹以后共享同套 runner，不用各自搭环境喵。

决定的前提是平台、规则、账号都会变，提前把路铺好，比被迫迁移省事。去中心化的价值是把平台风险变成自己可以决定的风险——想换 runner、换域名、把 repo 权限交给他人，都不需要平台点头喵。

推送成功后有种踏实感——不是工具通了，而是控制感回来了。日志已在新链路跑，明天继续调 CI 喵。
